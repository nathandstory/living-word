# Security Policy

Living Word streams scripture audio from configured remote providers and caches files locally. Security reports should focus on crashes, unsafe file handling, network handling, path traversal, or server/client desync risks.

## Supported Versions

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |

## Reporting A Vulnerability

Please report security-sensitive issues privately through GitHub's private vulnerability reporting if enabled, or by contacting the project maintainer directly.

Do not publicly post exploit details until the issue has been reviewed.
